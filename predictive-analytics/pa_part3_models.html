Залишки часового ряду — це не просто «шум». За ними ховається структура: автокореляція, стаціонарність, пам'ять. AR, MA, ARMA та ARIMA — мова, якою ця структура описується й використовується для прогнозування.

Моделі залишків — випадкові процеси

Після виокремлення тренду і сезонності залишається компонента, яку ми умовно назвали «шумом». Але навіть у цьому «шумі» часто є структура — автокореляція між сусідніми значеннями. Розбираємо математичний апарат дискретних випадкових процесів, АКФ і ЧАКФ, і головне поняття курсу — стаціонарність.

Дискретний випадковий процес

Залишки часового ряду — не детерміновані числа, а реалізація випадкового процесу. Дискретний випадковий процес — це сукупність випадкових величин {εₜ, t ∈ ℤ}, де кожна εₜ описується своїм розподілом, але між різними εₜ існує статистична залежність.

Аналогія

Уявіть температуру повітря кожного дня протягом року. Це не окремі незалежні вимірювання: холодна п'ятниця з більшою ймовірністю означає холодну суботу, ніж жарку. Послідовність таких залежних випадкових величин і є випадковим процесом. Одна конкретна реалізована послідовність температур — це часовий ряд.

WN
Білий шум (White Noise)

Некорельований процес: E[εₜ]=0, Var[εₜ]=σ², Cov(εₜ, εₛ)=0 при t≠s. Ідеальний залишок — ознака, що модель захопила весь сигнал.

WGN
Гаусів білий шум

Білий шум із нормальним розподілом εₜ ~ N(0, σ²). Незалежні (не просто некорельовані). Основа більшості параметричних моделей часових рядів.

RW
Випадкове блукання (Random Walk)

Yₜ = Yₜ₋₁ + εₜ. Кожний крок — незалежне збурення. Нестаціонарний. Класична модель цін на фінансових ринках. Дисперсія зростає лінійно з часом: Var[Yₜ] = t·σ².

Білий шум (стаціонарний) vs випадкове блукання (нестаціонарне)

Ергодичність — чому одна реалізація достатня

Стохастичний процес — це ансамбль нескінченно багатьох можливих реалізацій. На практиці ми маємо лише одну — один спостережуваний ряд. Як тоді оцінювати μ, γ(k)? Відповідь — ергодичність.

Ергодичність за середнім
Часове середнє: T̄(y) = (1/T) Σₜ₌₁ᵀ yₜ
Ансамблеве середнє: E[yₜ] = μ

Ергодичний процес: T̄(y) → μ при T → ∞ (зб. майже напевно)
Ергодичний процес: часова статистика (середня по одному довгому ряду) збігається з ансамблевою (середня по багатьох незалежних реалізаціях). Стаціонарність необхідна, але не достатня. Практичний наслідок: ми маємо право оцінювати АКФ і параметри ARIMA з одного ряду RetailPulse, бо він ергодичний — довгий ряд «обходить» всі типові стани процесу.
Аналогія: кидання кубика

Кинути один кубик 600 разів і усереднити (часова статистика) — майже те саме, що кинути 600 різних кубиків одночасно (ансамблева статистика). Для ергодичного процесу «один ряд, але довгий» еквівалентний «багатьом рядам, але коротким». Саме ця властивість дозволяє нам будувати SARIMA для RetailPulse, маючи лише 3-річну історію продажів — замість паралельного запуску 1000 незалежних мереж.

Процес Гаусса–Маркова

Між білим шумом (без пам'яті) і загальними ARMA-моделями є важливий клас: процеси Гаусса–Маркова — гауссівські процеси з «марківською властивістю», де прогноз визначається лише поточним станом.

Процес Гаусса–Маркова першого порядку
yₜ = φ·yₜ₋₁ + εₜ, εₜ ~ N(0, σ²), |φ| < 1

E[yₜ₊₁ | y₀, y₁, ..., yₜ] = E[yₜ₊₁ | yₜ] = φ·yₜ ← марківська властивість
γ(k) = σ²·φᵏ / (1 − φ²) ← АКФ спадає як φᵏ
Марківська властивість: знаючи поточний стан yₜ, минуле не дає жодної додаткової інформації для прогнозу. Це AR(1) з нормальним шумом. При φ=0 — білий шум. При |φ|=1 — випадкове блукання (нестаціонарне). При |φ|<1 — стаціонарний процес із поверненням до середнього (mean reversion).
|φ|<1
Стаціонарний GM

Процес повертається до середнього (mean reversion). АКФ спадає геометрично як φᵏ. Найпростіший стаціонарний процес із «пам'яттю» — основа AR(1) і фундаментальний будівельний блок усієї класи ARMA.

φ=1
Випадкове блукання

Вироджений граничний випадок GM. Немає повернення до середнього. Дисперсія зростає лінійно з часом. Нестаціонарний — потрібне диференціювання (d=1 в ARIMA).

AR(p)
Узагальнення

GM вищого порядку відповідає AR(p). Стаціонарність забезпечується, коли всі корені характеристичного рівняння лежать поза одиничним колом. Це теж стосується повного класу причинних ARMA-моделей.

Автокореляційна функція (АКФ)

АКФ вимірює статистичний зв'язок між значеннями часового ряду, зсунутими на k кроків відносно один одного. Це головний інструмент для виявлення структури у залишках і вибору порядку ARMA-моделі.

Функція автоковаріації та АКФ
γ(k) = Cov(yₜ, yₜ₋ₖ) = E[(yₜ − μ)(yₜ₋ₖ − μ)]
ρ(k) = γ(k) / γ(0) ∈ [−1, +1] ← АКФ при лагу k

Оцінка: r(k) = Σ(yₜ − ȳ)(yₜ₋ₖ − ȳ) / Σ(yₜ − ȳ)²
ρ(0) = 1 завжди (кореляція з самим собою). Для білого шуму ρ(k) ≈ 0 при k > 0. Якщо ACF повільно спадає — ряд нестаціонарний. Якщо ACF обривається після лагу q — натяк на MA(q). Якщо поступово спадає — натяк на AR(p).
АКФ (ACF)
Автокореляційна функція

Пряма кореляція yₜ та yₜ₋ₖ. Включає як прямий, так і непрямий вплив. Використовується для визначення порядку q в MA-моделях: ACF «обривається» на лагу q.

ЧАКФ (PACF)
Часткова автокореляційна функція

Кореляція yₜ та yₜ₋ₖ після виключення впливу проміжних лагів. Чистий прямий ефект лагу k. Використовується для визначення порядку p в AR-моделях: PACF «обривається» на лагу p.

ЧАКФ — формальне визначення через проекцію
ЧАКФ(k) = φₖₖ — k-й коефіцієнт AR(k)-моделі (YW порядку k): yₜ = φₖ₁yₜ₋₁ + φₖ₂yₜ₋₂ + ... + φₖₖyₜ₋ₖ + εₜ Тобто ЧАКФ(k) = Cor(yₜ − Proj(yₜ | yₜ₋₁,...,yₜ₋ₖ₊₁), yₜ₋ₖ − Proj(yₜ₋ₖ | yₜ₋₁,...,yₜ₋ₖ₊₁)) Смуга значущості: |ЧАКФ(k)| > 1.96/√n ← 95% рівень
Інтерпретація через YW: якщо підібрати AR(k) методом Юла–Вокера, останній коефіцієнт φₖₖ і є ЧАКФ(k). Тому ЧАКФ = «діагональ» матриці YW-оцінок при зростаючому порядку. Для AR(p): ЧАКФ(k)≠0 при k≤p і ЧАКФ(k)≈0 при k>p — ось чому ЧАКФ «обривається» на лагу p.

Типові патерни АКФ і ЧАКФ

Стаціонарність — фундаментальна властивість

Більшість методів аналізу часових рядів (ARMA, спектральний аналіз) вимагають стаціонарності. Строга стаціонарність складна для перевірки; зазвичай перевіряють слабку (коваріаційну) стаціонарність:

Умови слабкої стаціонарності
1) E[yₜ] = μ = const (постійне середнє)
2) Var[yₜ] = σ² < ∞ = const (постійна дисперсія)
3) Cov(yₜ, yₜ₊ₖ) = γ(k) (залежить лише від лагу k, не від t)
Ряд із трендом нестаціонарний (порушення умови 1). Ряд із сезонністю нестаціонарний (порушення умов 1 і 2). Випадкове блукання нестаціонарне (дисперсія зростає). Для застосування ARMA потрібно спочатку «зробити» ряд стаціонарним — зазвичай через диференціювання.
Як перевірити стаціонарність на практиці

Три підходи: (1) Візуально — чи є очевидний тренд або зростаюча/спадаюча дисперсія на графіку? (2) ACF — якщо повільно спадає без чіткого відсікання, ряд нестаціонарний. (3) Тест Дікі–Фуллера (ADF test, statsmodels): нульова гіпотеза — наявність одиничного кореня (нестаціонарність). Якщо p-value < 0.05 — відкидаємо H₀ і вважаємо ряд стаціонарним.

Детерміністичний vs стохастичний тренд

Нестаціонарність буває двох типів — і вибір методу усунення кардинально різниться:

Детерміністичний тренд (DS)
yₜ = α + βt + εₜ

Тренд — фіксована функція часу. Шоки мають тимчасовий вплив: ряд завжди повертається до лінії тренду. Лікування: видалення тренду (detrending) — OLS-регресія на t, залишки стаціонарні.

Стохастичний тренд (DS / одиничний корінь)
yₜ = yₜ₋₁ + εₜ (RW)

Тренд «вмонтований» у структуру ряду. Шоки мають постійний вплив: ряд ніколи не повертається до попереднього рівня. Лікування: диференціювання — ∇yₜ = yₜ − yₜ₋₁ стаціонарне.

Аналогія: гумка vs. п'яний матрос

Детерміністичний тренд — як м'яч на гумці, прив'язаній до лінії: скільки б його не відкинути — він повернеться. Стохастичний тренд (RW) — як п'яний матрос: кожен крок випадковий, і попередньої «якірної точки» немає. Якщо ви по-помилку диференціюєте DS-ряд, отримаєте MA(1) з від'ємним коренем — «зайве» диференціювання (overdifferencing).

Overdifferencing — як розпізнати зайве диференціювання

Симптом: після диференціювання MA(q)-компонента у залишках має коефіцієнт θ₁ ≈ −0.9..−1.0. Це ознака того, що ряд був уже стаціонарним до диференціювання (або DS, а не SS), і d обрано надмірно. Наслідки: модель перепараметризована, стандартні похибки роздуті, AIC гірший. Виправлення: спробуйте d−1 і порівняйте AIC. Якщо KPSS приймає H₀ (ряд стаціонарний) вже при d=0 — диференціювання взагалі не потрібне. Правило: якщо ACF після Δ¹yₜ виглядає добре, не диференціюйте двічі без підстав.

Тест KPSS (Kwiatkowski–Phillips–Schmidt–Shin)
H₀ (KPSS): ряд стаціонарний (немає одиничного кореня) H₁ (KPSS): ряд нестаціонарний Статистика: η̂ = (1/T²) · Σₜ Sₜ² / σ̂² де Sₜ = Σⱼ₌₁ᵗ ê_j — часткові суми OLS-залишків Критичні значення (рівень): 0.10 → 0.347 | 0.05 → 0.463 | 0.01 → 0.739 Якщо η̂ > крит. знач. — відкидаємо H₀ (ряд нестаціонарний)
KPSS і ADF — взаємодоповнюючі: ADF (H₀ = нестаціонарний), KPSS (H₀ = стаціонарний). Якщо обидва «погоджуються» → надійний висновок. Якщо суперечать → можливий тренд-стаціонарний процес або структурний злам. Використовуйте разом: statsmodels.tsa.stattools.kpss.
Q&A
Питання для самоперевірки
01Чим відрізняється «некорельованість» від «незалежності» для WN vs WGN?
+
Некорельованість означає відсутність лінійного зв'язку між εₜ та εₛ (при t≠s). Незалежність — сильніша умова: жодного статистичного зв'язку (лінійного чи нелінійного). Для гаусівського білого шуму (WGN) некорельованість ↔ незалежність, бо для нормального розподілу нульова коваріація еквівалентна незалежності. Для негаусівського WN це не так: можуть бути некорельовані, але залежні (наприклад, через нелінійний зв'язок — квадратична залежність).
02Чому ACF поступово спадає для AR-процесу, але різко обривається для MA?
+
AR(p): кожне значення залежить від p попередніх, але через ланцюг «yₜ → yₜ₋₁ → yₜ₋₂ → ...» непряма кореляція передається нескінченно (поступово загасаючи). Тому ACF має «хвіст» без різкого відсікання. MA(q): кожне значення залежить лише від q попередніх шумів εₜ. Якщо k > q — прямого спільного шуму між yₜ та yₜ₋ₖ немає → ρ(k)=0 для k>q. Тому ACF різко «обривається» на лагу q. Це «правило АКФ–ЧАКФ» — головний інструмент вибору p і q.
03Ціна акції — це випадкове блукання. Що це означає для прогнозування?
+
Якщо ціна є Random Walk: Pₜ = Pₜ₋₁ + εₜ, то найкращий прогноз на будь-який горизонт — це поточна ціна. Дисперсія прогнозу зростає лінійно з горизонтом, а кожен поступальний крок непередбачуваний. Це і є гіпотеза ефективного ринку (Fama): жодна стратегія не може стабільно «бити» ринок, бо всі доступні публічно новини вже враховані в поточній ціні. Практичний висновок: для RW ARMA-моделі не дають кращого прогнозу, ніж наївний «завтра = сьогодні».
04Що таке ергодичність і чому вона дозволяє оцінювати АКФ з єдиного ряду?
+
Процес ергодичний, якщо часові середні збігаються з ансамблевими (по множині паралельних реалізацій). Формально: (1/T)·Σyₜ → E[yₜ] при T→∞. Без ергодичності статистика одного ряду не оцінює параметри розподілу — потрібні сотні незалежних реалізацій. На практиці нестаціонарний процес (детермінований тренд, RW) не є ергодичним: часове середнє дрейфує і не збігається до константи. Після стаціонаризації (диференціювання, видалення тренду) процес стає ергодичним, і ми можемо надійно оцінити γ(k) = (1/T)·Σyₜ·yₜ₋ₖ з єдиного спостережуваного ряду. Саме тому перевірка стаціонарності — перший обов'язковий крок у методології Бокса–Дженкінса.

AR, MA, ARMA, ARIMA

Клас ARMA-моделей — це ключовий інструментарій аналізу стаціонарних часових рядів. Авторегресійна (AR) компонента описує, як минулі значення ряду впливають на поточне. Компонента ковзного середнього (MA) — як минулі шуми впливають на поточне. Разом вони утворюють ARMA, а додавання диференціювання для нестаціонарних рядів дає ARIMA.

Оператор зсуву B (back-shift / lag operator)

Перш ніж переходити до ARMA-моделей, введемо зручне позначення — оператор зсуву B. Він робить запис складних моделей компактним і дозволяє маніпулювати ними як звичайними многочленами.

Визначення та властивості оператора B
Визначення: B·yₜ = yₜ₋₁ (зрушує ряд на 1 крок назад) Степені: B²·yₜ = yₜ₋₂, Bᵏ·yₜ = yₜ₋ₖ B⁰·yₜ = yₜ (одиничний оператор) Різниця першого порядку: (1 − B)·yₜ = yₜ − yₜ₋₁ = Δyₜ d-та різниця: (1 − B)ᵈ·yₜ = Δᵈyₜ Сезонна різниця (лаг m): (1 − Bᵐ)·yₜ = yₜ − yₜ₋ₘ = Δₘyₜ AR(p) через B: (1 − φ₁B − φ₂B² − ... − φₚBᵖ)·yₜ = εₜ MA(q) через B: yₜ = (1 + θ₁B + θ₂B² + ... + θ_qBq)·εₜ ARMA(p,q): φ(B)·yₜ = θ(B)·εₜ ARIMA(p,d,q): φ(B)·(1−B)ᵈ·yₜ = θ(B)·εₜ
Поліноми φ(B) і θ(B) — це звичайні многочлени від «аргументу B». Правила алгебри зберігаються: φ(B)·θ(B) = добуток многочленів, і так далі. Завдяки цьому SARIMA у вигляді φ(B)Φ(Bᵐ)(1−B)ᵈ(1−Bᵐ)ᴰyₜ = θ(B)Θ(Bᵐ)εₜ читається як звичайне алгебраїчне рівняння над B.

Авторегресійна модель AR(p)

AR(p) стверджує: поточне значення ряду є лінійною комбінацією p попередніх значень плюс білий шум. Найпростіша і найінтуїтивніша модель — «завтра залежить від вчора».

Авторегресійна модель AR(p)
yₜ = φ₁yₜ₋₁ + φ₂yₜ₋₂ + ... + φₚyₜ₋ₚ + εₜ

AR(1): yₜ = φ·yₜ₋₁ + εₜ
Умова стаціонарності: |φ| < 1 (для AR(1))
Параметри φᵢ оцінюються методом найменших квадратів або MLE. При |φ| = 1 — одиничний корінь (Random Walk). При |φ| > 1 — ряд вибухово нестабільний. Для AR(p) умова стаціонарності: всі корені характеристичного полінома 1 − φ₁z − ... − φₚzᵖ = 0 за модулем > 1.
Аналогія AR(1) для продажів

Якщо φ=0.8, то: «Якщо цього тижня продали на 10 тис. більше за середнє, наступного тижня (в середньому) продамо на 8 тис. більше за середнє». Потім ефект затухає: через тиждень — +6.4 тис., ще через — +5.1 тис. AR(1) описує «пам'ять» ринку — попит не стрибає хаотично, а має інерцію.

Рівняння Юла–Вокера (Yule–Walker)
Для AR(p) ковариаційна структура підпорядковується системі: ρ(k) = φ₁·ρ(k−1) + φ₂·ρ(k−2) + ... + φₚ·ρ(k−p), k = 1, 2, ..., p У матричному вигляді: R·φ = ρ де R = [ρ(i−j)]ᵢ,ⱼ — матриця АКФ (розмір p×p) φ = (φ₁, ..., φₚ)ᵀ — вектор AR-коефіцієнтів ρ = (ρ(1), ..., ρ(p))ᵀ — вектор значень АКФ Розв'язок: φ̂ = R⁻¹·ρ̂ (оцінки через вибіркову АКФ)
Практичне значення: (1) Юла–Вокер пов'язує AR-параметри напряму з АКФ — тому АКФ/ЧАКФ є діагностичним інструментом. (2) Оцінки YW не потребують ітеративної оптимізації — просто лінійна система. (3) ЧАКФ при лагу k — це коефіцієнт φₖ у Yule-Walker системі порядку k (тому ЧАКФ = PACF «обривається» на справжньому порядку AR). В Python: statsmodels.regression.linear_model.yule_walker(y, order=p).

Модель ковзного середнього MA(q)

MA(q) стверджує: поточне значення залежить від q попередніх шокових впливів (шумів), а не від попередніх значень самого ряду. Менш інтуїтивна, але дуже ефективна на практиці.

Модель ковзного середнього MA(q)
yₜ = εₜ + θ₁εₜ₋₁ + θ₂εₜ₋₂ + ... + θqεₜ₋q

MA(1): yₜ = εₜ + θ·εₜ₋₁
Умова оберненості: |θ| < 1 (для MA(1))
MA(q) завжди стаціонарна (якщо виконана умова оберненості). АКФ MA(q) дорівнює нулю для лагів k > q — «обривається» на q. Це ключова ознака для діагностики. Умова оберненості потрібна для існування еквівалентного AR(∞)-представлення.
Аналогія MA для магазинних залишків

Уявіть, що на полицях раптово закінчився популярний товар (шок εₜ). Це вплине на продажі наступних двох тижнів, поки завезуть нову партію (θ₁, θ₂ ≠ 0). Але після двох тижнів (q=2) ефект зникне — продажі повернуться до нормального. MA(q) якраз описує ефекти короткострокових шоків з обмеженою тривалістю.

Загальна модель ARMA(p, q)

Об'єднуємо AR і MA в одній моделі: ряд залежить і від своїх минулих значень, і від минулих шоків. Більша гнучкість при меншій кількості параметрів.

ARMA(p, q)
yₜ = φ₁yₜ₋₁ + ... + φₚyₜ₋ₚ + εₜ + θ₁εₜ₋₁ + ... + θqεₜ₋q

У компактній формі через оператор зсуву B (By_t = y_{t-1}):
Φ(B)·yₜ = Θ(B)·εₜ
Принцип ощадливості (parsimony): ARMA(1,1) з двома параметрами часто краще підходить до реальних даних, ніж AR(8) або MA(8) — бо пояснює схожу автокореляційну структуру меншою кількістю параметрів і менше перенавчається.

ARIMA(p, d, q) — для нестаціонарних рядів

ARIMA — Autoregressive Integrated Moving Average. «I» — «Integrated» — позначає d-кратне диференціювання, яке перетворює нестаціонарний ряд на стаціонарний перед застосуванням ARMA.

ARIMA(p, d, q)
wₜ = Δᵈyₜ — d-кратна різниця
ARMA(p, q) застосовується до wₜ

ARIMA(1,1,0): wₜ = yₜ − yₜ₋₁ = φ·wₜ₋₁ + εₜ
ARIMA(0,1,0): wₜ = εₜ → Random Walk (якщо d=1)
d=1 усуває лінійний тренд; d=2 — квадратичний. Для більшості реальних рядів d=1 або d=2 достатньо. SARIMA(p,d,q)(P,D,Q)[m] розширює модель, додаючи AR, I, MA компоненти для сезонного лагу m — найбільш практична модель для RetailPulse.

SARIMA — сезонне розширення ARIMA

SARIMA(p,d,q)(P,D,Q)[m] — найпрактичніша модель для реальних рядів із сезонністю. Вона складається з двох шарів: короткострокові оператори (p,d,q) і сезонні (P,D,Q) з кроком m лагів.

SARIMA(p,d,q)(P,D,Q)[m] — повний запис
φ(B)·Φ(Bᵐ)·(1−B)ᵈ·(1−Bᵐ)ᴰ·yₜ = θ(B)·Θ(Bᵐ)·εₜ

ARIMAX — ARIMA з екзогенними змінними

ARIMA описує ряд лише через його власну історію. Але що, якщо відомі зовнішні чинники (ціни, свята, погода), які впливають на попит? ARIMAX (ARIMA with eXogenous inputs) додає регресійну складову на зовнішні змінні:

ARIMAX(p, d, q) — регресія з ARIMA-помилками
φ(B)·∇ᵈyₜ = β₁x₁ₜ + β₂x₂ₜ + ... + βₖxₖₜ + θ(B)·εₜ Або еквівалентно: yₜ = β·Xₜ + ηₜ, де ηₜ = φ(B)⁻¹θ(B)·εₜ — ARMA-помилки xᵢₜ — екзогенні змінні (можуть бути: індикатори свят, ціна, акційна знижка, середня температура, dummy-злам) βᵢ — коефіцієнти регресії (оцінюються спільно з φ, θ) εₜ — WN (стандартний шум моделі)
Відмінність від звичайної регресії: тут помилки ηₜ — не WN, а ARMA-процес. Ігнорування цього (звичайний OLS) дає зміщені стандартні похибки та хибні t-тести. Правильна оцінка: MLE або умовний МНК. В Python: statsmodels.tsa.statespace.SARIMAX(y, exog=X) — той самий клас SARIMAX приймає exog-матрицю X.
Коли додавати екзогенні змінні

Є чітка бізнес-логіка: «у дні акцій попит зростає». Є достатньо даних. Зовнішня змінна доступна на горизонті прогнозу (або прогнозується окремо). АКФ залишків ARIMA показує систематичну структуру → можливо, пропущена змінна.

Якщо xₜ₊ₕ невідома (наприклад, завтрашня температура), прогноз ŷₜ₊ₕ є умовним: «якщо температура = 15°C, то продажі будуть Х». Щоб отримати безумовний прогноз, треба спочатку прогнозувати xₜ₊ₕ окремою моделлю — і тоді похибка прогнозу X накопичується разом із похибкою y. Лінійні процеси і теорема Вольда

AR, MA та ARMA є частковими випадками ширшого класу — лінійних процесів. Розуміння цього класу пояснює, чому ARMA покриває практично всі стаціонарні ряди.

Лінійний процес (MA(∞) представлення)
yₜ = μ + Σⱼ₌₀^∞ ψⱼ · εₜ₋ⱼ, εₜ ~ WN(0, σ²)

ψ₀ = 1 (нормування), Σψⱼ² < ∞ (квадратична сумовність)

γ(k) = σ² · Σⱼ₌₀^∞ ψⱼ · ψⱼ₊ₖ ← АКФ через ψ-ваги
Будь-який стаціонарний ARMA(p,q) має MA(∞) представлення через рекурентну заміну AR-частини. Коефіцієнти ψⱼ — impulse response (імпульсна відповідь): вплив одиничного шоку εₜ через j кроків. Спадання ψⱼ→0 означає «загасаючу пам'ять» моделі.
Теорема Вольда (декомпозиція Уолда)

Будь-який стаціонарний процес можна точно представити як суму: (1) детермінованих (абсолютно передбачуваних) компонент + (2) лінійного процесу MA(∞). Лінійна частина — саме те, що ARMA(p,q) апроксимує скінченною кількістю параметрів. Звідси принцип парсимонії: шукаємо найменшу модель із достатнім p і q, яка вловлює всю лінійну структуру даних.

Причинність і оборотність ARMA-моделей

Не кожна ARMA-специфікація практично корисна. Дві ключові умови — причинність (causal) і оборотність (invertible) — визначають, чи придатна модель для оцінки та прогнозування.

Умови причинності та оборотності
AR-поліном: φ(z) = 1 − φ₁z − φ₂z² − ... − φₚzᵖ
MA-поліном: θ(z) = 1 + θ₁z + θ₂z² + ... + θqzq

Причинність: всі корені φ(z) = 0 лежать ПОЗА одиничним колом |z| > 1
Оборотність: всі корені θ(z) = 0 лежать ПОЗА одиничним колом |z| > 1
AR(1): yₜ = φyₜ₋₁ + εₜ → причинний ↔ |φ| < 1. MA(1): yₜ = εₜ + θεₜ₋₁ → оборотний ↔ |θ| < 1. Якщо причинність порушена — AR-частина нестаціонарна. Якщо оборотність порушена — MA неідентифікована (дві різні моделі мають однакову АКФ).
Causal
Причинність

Причинний процес: yₜ залежить лише від минулих шоків εₜ₋ⱼ (j≥0), а не майбутніх. Забезпечує збіжне MA(∞) представлення із ψⱼ→0. AR(1) причинний тоді і тільки тоді, коли |φ|<1.

Inv
Оборотність

Оборотна MA-модель: шок εₜ виражається через минулі yₜ (AR(∞) представлення). Без цього MLE не може відновити εₜ із спостережень — оцінювання некоректне. MA(1) оборотний ↔ |θ|<1.

Ident
Ідентифікованість

MA(1) з θ і MA(1) з 1/θ мають однакову АКФ — вони неідентифіковані між собою. Оборотність усуває цю неоднозначність: завжди беремо канонічне представлення з |θ|<1.

Ідентифікація моделі: методологія Бокса–Дженкінса

Найпопулярніший систематичний підхід до вибору p, d, q — покрокова методологія Бокса–Дженкінса. Ключові правила:

  • Крок 1 — Стаціонаризація: візуально і ADF-тест. Якщо нестаціонарний — диференціюємо (d=1, потім d=2 за потреби)
  • Крок 2 — Визначення p і q: аналіз АКФ (дає q) і ЧАКФ (дає p) стаціонарного ряду
  • Крок 3 — Оцінка: MLE або умовний МНК для кількох кандидатів
  • Крок 4 — Діагностика залишків: залишки мають бути білим шумом (ADF, Ljung-Box тест, ACF залишків)
  • Крок 5 — Вибір кращої моделі: AIC або BIC (штрафують за складність). Менший — краще
Критерії вибору моделі: AIC і BIC
AIC = −2·ℓ + 2·k
BIC = −2·ℓ + k·ln(n)

ℓ — log-likelihood оціненої моделі
k — кількість параметрів (p + q + P + Q + константа)
n — кількість спостережень
Менший AIC/BIC — краща модель. BIC штрафує сильніше (ln(n) > 2 при n > 7), тому частіше обирає простіші моделі. Практичне правило: якщо ΔAIC < 2 між двома моделями — різниця несуттєва, обирають простішу. AIC краще для прогнозу; BIC — для інтерпретації та пошуку «справжньої» структури.

Інтервали прогнозування

Точковий прогноз без інтервалу — лише половина роботи. Інтервал прогнозування показує діапазон, де з заданою ймовірністю опиниться реальне значення. Він принципово ширший за довірчий інтервал середнього.

95% інтервал прогнозування ARIMA(p,d,q)
ŷₜ₊ₕ ± 1.96 · σ̂ₕ

σ̂²ₕ = σ²ε · Σⱼ₌₀ʰ⁻¹ ψⱼ² ← дисперсія h-крокового прогнозу

ψⱼ — impulse response коефіцієнти (MA(∞) представлення)
h = 1: σ̂² = σ²ε (мінімум — тільки один крок)
h → ∞: σ̂²ₕ → Var[yₜ] (насичення на дисперсії ряду)
Ширина конусу зростає з горизонтом — це не дефект моделі, а математична властивість. Для RetailPulse (h=14 тижнів): якщо σε = 50 тис. грн і ψ₁≈0.8, ψ₂≈0.6, ..., інтервал на h=14 може бути ±300 тис. Показуйте інтервал замовнику завжди — він визначає стратегію замовлення товару.

Структурні зламини — коли ряд «змінює правила»

Структурний злам (structural break) — це момент часу, в якому параметри моделі різко змінюються. Це не аномалія-одиночка, а зміна режиму. Ігнорування зламу: модель намагається підібрати середнє між двома режимами — і не описує жодного коректно.

Тест Чоу (Chow test)

Перевірка злому в заздалегідь відомій точці τ. Порівнює RSS повної моделі (один ARMA на всі дані) з RSS двох субмоделей (до τ і після τ). Статистика F ~ F(k, n−2k). Недолік: треба знати τ заздалегідь.

Тест Квятковського–Перрона (sup-F / CUSUM)

Тест на злам у невідомій точці: перебирає всі τ і обирає той, де статистика F максимальна. CUSUM-тест відстежує кумулятивну суму залишків рекурсивної регресії — відхилення від горизонталі вказує на злам.

Попит на окремі категорії (санітайзери, консерви) зрісна 5σ за два тижні. Рішення: (1) окремі моделі «до» та «після», (2) dummy-змінна в ARIMAX, (3) rolling window — стара передісторія не бере участі у навчанні.

Структурний злам ≠ викид

Викид (outlier) — поодинока точка, яка не впливає на подальші значення; після неї ряд повертається до попереднього рівня. Структурний злам — зміна генеруючого процесу: середнє, тренд або параметри ARMA назавжди змінились. Лікування різне: викид — заміна/коректура точки; злам — розділення вибірки або dummy-змінна.

Комбінування прогнозів — мудрість натовпу

Жодна окрема модель не є оптимальною за всіх умов. Комбінування (forecast combination) знижує варіансу і захищає від помилки вибору моделі:

Комбінований прогноз
ŷₜ₊ₕᶜᵒᵐᵇ = Σᵢ wᵢ · ŷₜ₊ₕ⁽ⁱ⁾ Simple Average: wᵢ = 1/M (M — кількість моделей) Variance-based: wᵢ ∝ 1/σᵢ² (менша помилка → більша вага) OLS combination: w* = argmin Σₜ (yₜ − Σᵢ wᵢŷₜ⁽ⁱ⁾)²
Теорема Бейтса–Грейнджера (1969): комбінований прогноз завжди має дисперсію похибки, не гіршу за найкращу окрему модель (за умови некорельованості похибок). На практиці: просте середнє часто перемагає складні схеми зважування.

Q&A
Питання для самоперевірки
01Як за графіками ACF і PACF розпізнати AR(2) процес?
+
Для AR(2): ACF поступово спадає (не обривається — можуть бути осциляції при від'ємних φ). PACF обривається після лагу 2 — значущі тільки лаги 1 і 2, всі подальші статистично незначущі (у межах ±1.96/√n). Ключове правило: якщо PACF «обривається» на лагу p — розглядаємо AR(p).
02Що таке умова причинності (causality) для AR-моделі і навіщо вона потрібна?
+
Умова причинності для AR(p): всі корені характеристичного полінома φ(z) = 1 − φ₁z − ... − φₚzᵖ лежать поза одиничним колом (|zᵢ| > 1). Вона гарантує, що yₜ залежить тільки від минулого (yₜ₋₁, yₜ₋₂,...) і теперішнього шуму — не від майбутнього. Якщо умова порушена — модель «нефізична»: майбутнє «пояснює» минуле. Практично перевіряють: statsmodels після оцінки виводить warning, якщо корені близькі до одиничного кола.
03AIC vs BIC — коли використовувати який критерій вибору моделі?
+
AIC (Akaike Information Criterion) мінімізує помилку прогнозу на нових даних — схильний обирати складніші моделі. BIC (Bayesian IC) додає більший штраф за параметри: більш суворий і схильний до простоти. Правило: якщо мета — якнайточніший прогноз і вибірка велика — AIC. Якщо мета — інтерпретабельна, «справжня» модель або вибірка мала — BIC. Для RetailPulse з великим обсягом даних — AIC. Для коротких рядів по окремих SKU — BIC.
04Чому залишки хорошої ARIMA-моделі мають бути «білим шумом»?
+
Якщо залишки мають автокореляцію — модель не захопила весь лінійний сигнал у даних. Цю автокореляцію можна було б використати для кращого прогнозу, але ми цього не зробили. Тест Ljung-Box перевіряє, чи є в залишках значуща автокореляція (H₀: залишки — WN). Якщо p-value < 0.05 — залишки не є WN і модель неповна. Крім того, залишки «WN» — умова коректності довірчих інтервалів прогнозу.

def grid_search_sarima(series, pdq, PDQ, m=52):
    """Підбір SARIMA(p,d,q)(P,D,Q)[m] за мінімальним AIC"""

            continue
    return best

import numpy as np
from statsmodels.tsa.statespace.sarimax import SARIMAX

def walk_forward(series, train_size, order, seasonal_order, horizon=1):
    """Walk-forward (expanding window) — найчесніша оцінка моделі"""
    preds, actuals = [], []
    for i in range(train_size, len(series) - horizon + 1):
        tr = series.iloc[:i]
        try:
            res = SARIMAX(tr, order=order,
                          seasonal_order=seasonal_order).fit(disp=False)
            fc  = res.forecast(horizon)
            preds.append(float(fc.iloc[-1]))
            actuals.append(float(series.iloc[i + horizon - 1]))
        except Exception:
            continue

    y, p = np.array(actuals), np.array(preds)
    return {
        'mape': np.mean(np.abs((y - p) / y)) * 100,
        'mae':  np.mean(np.abs(y - p)),
        'n':    len(y)
    }

lb = acorr_ljungbox(resid, lags=[10, 20], return_df=True)

Шпаргалка формул

Ключові формули всього курсу PA & TSA — для швидкого пошуку під час роботи з даними.

M1–M2 · Метрики точності
MAE = (1/n)·Σ|yₜ − ŷₜ|
RMSE = √(1/n·Σ(yₜ − ŷₜ)²)
MAPE = (100/n)·Σ|eₜ/yₜ|
sMAPE = (200/n)·Σ|eₜ|/(|yₜ|+|ŷₜ|)
MASE = MAE / (1/(n−1)·Σ|yₜ−yₜ₋₁|)
M1 · Bias–Variance
MSE = Bias² + Variance + σ²
Bias = E[ŷ] − y
Variance = E[(ŷ − E[ŷ])²]
MLE: θ̂ = argmax Σ ln p(yₜ|θ)
MAP: θ̂ = argmax [ln L + ln p(θ)]
M2 · Декомпозиція ряду
Адитивна: yₜ = Tₜ + Sₜ + Cₜ + Rₜ
Мульт.: yₜ = Tₜ × Sₜ × Cₜ × Rₜ
ACF: ρ(k) = γ(k)/γ(0)
γ(k) = Cov(yₜ, yₜ₋ₖ)
95% межі: ±1.96/√n
M3 · SES та Holt
SES: ŷₜ₊₁ = α·yₜ + (1−α)·ŷₜ
Holt рівень: lₜ = α·yₜ + (1−α)(lₜ₋₁+bₜ₋₁)
Holt тренд: bₜ = β·(lₜ−lₜ₋₁) + (1−β)·bₜ₋₁
Прогноз: ŷₜ₊ₕ = lₜ + h·bₜ
Damped: ŷₜ₊ₕ = lₜ + (Σᵢ₌₁ʰ φⁱ)·bₜ
M3 · Holt–Winters (адитивний)
lₜ = α(yₜ − sₜ₋ₘ) + (1−α)(lₜ₋₁ + bₜ₋₁)
bₜ = β(lₜ − lₜ₋₁) + (1−β)bₜ₋₁
sₜ = γ(yₜ − lₜ₋₁ − bₜ₋₁) + (1−γ)sₜ₋ₘ
ŷₜ₊ₕ = lₜ + h·bₜ + sₜ₋ₘ₊ₕₘₒ꜀
Box-Cox: y*(λ) = (yλ−1)/λ; ln(y) якщо λ=0
M4 · SSA (Гусениця)
Траєкторна матриця: X (L×K), K=N−L+1
SVD: X = U·Σ·Vᵀ = Σᵢ √λᵢ·Uᵢ·VᵢᵀС
Реконструкція r-SSA: X̃ = Σᵢ₌₁ʳ √λᵢ·Uᵢ·Vᵢᵀ
Рекурент. прогноз: yₙ₊ₕ = Σᵢ₌₁ᴸ⁻¹ aᵢ·yₙ₊ₕ₋ᵢ
aᵢ = (Σⱼ₌₁ʳ πⱼ·Uⱼ(i)·Uⱼ(L))/(1−ν²)
M5–M6 · Оператор зсуву
Byₜ = yₜ₋₁ ; Bᵏyₜ = yₜ₋ₖ
∇yₜ = (1−B)yₜ = yₜ − yₜ₋₁
∇ᵈyₜ = (1−B)ᵈyₜ
Сезонна різниця: ∇ₘyₜ = (1−Bᵐ)yₜ
AR-поліном: φ(B) = 1−φ₁B−⋯−φₚBᵖ
M5 · АКФ, ЧАКФ, Юла–Уокера
γ(k) = E[yₜ·yₜ₋ₖ] (стаціонарний)
ρ(k) = γ(k)/γ(0)
ЧАКФ φₖₖ: проекція yₜ на yₜ₋ₖ\span{yₜ₋₁,…,yₜ₋ₖ₊₁}
YW: R·φ = ρ → φ̂ = R⁻¹ρ̂
Rᵢⱼ = ρ(|i−j|), ρ = [ρ(1),…,ρ(p)]ᵀ
M6 · ARMA / ARIMA / SARIMA
AR(p): yₜ = φ₁yₜ₋₁+⋯+φₚyₜ₋ₚ + εₜ
MA(q): yₜ = εₜ + θ₁εₜ₋₁+⋯+θqεₜ₋q
ARMA: φ(B)yₜ = θ(B)εₜ
ARIMA: φ(B)∇ᵈyₜ = θ(B)εₜ
SARIMA: Φ(Bᵐ)φ(B)∇ᵈ∇ᴰₘyₜ = Θ(Bᵐ)θ(B)εₜ
M6 · Вибір моделі & діагностика
AIC = −2ln(L̂) + 2k
BIC = −2ln(L̂) + k·ln(n)
Ljung-Box: Q = n(n+2)·Σₖρ̂²(k)/(n−k)
Pred. interval: ŷₜ₊ₕ ± z·σₕ
σ²ₕ(AR1) = σ²·(1−φ²ʰ)/(1−φ²)
Таблиця вибору p, q
Процес АКФ ЧАКФ
AR(p) спадає відсік p
MA(q) відсік q спадає
ARMA(p,q) спадає спадає
WN ≈ 0 ≈ 0
M5 · Тести стаціонарності
ADF H₀: є одиничний корінь (нестац.)
ADF: τ = φ̂/SE(φ̂) → крит. знач. < −1.96
KPSS H₀: ряд стаціонарний
KPSS: η = (1/n²σ²)·ΣSₜ², Sₜ=Σᵢ₌₁ᵗ eᵢ
Нестац.: ADF не відх., KPSS відх.

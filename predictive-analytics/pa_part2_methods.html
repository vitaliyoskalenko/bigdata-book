Аналіз та оцінювання тренду

Тренд — найочевидніша компонента часового ряду, але правильно її виокремити і спрогнозувати — нетривіально. Проходимо шлях від найпростішого ковзного середнього через рекурентне оцінювання до повноцінних адаптивних моделей Холта та Холта–Вінтерса, які одночасно відстежують і тренд, і сезонність.

Задача оцінювання тренду

Формально часовий ряд представляють як суму детермінованої компоненти (тренд + сезонність) і випадкового шуму:

Базова модель тренду
yₜ = f(c, t) + εₜ
де f(c,t) — тренд (функція параметрів c та часу t)
εₜ — випадковий шум: E[εₜ] = 0, Var[εₜ] = σ²
Задача: оцінити c̃ₜ за наявними даними і потім обчислити прогноз ŷₜ₊d = f(c̃ₜ, t+d). Для цього використовують методи від класичного МНК до адаптивних рекурентних алгоритмів.

Ковзне середнє (Moving Average)

Найпростіший метод згладжування тренду. Ідея: замінити кожне значення середнім сусідніх значень у вікні ширини T. Метод чутливий до параметра — ширини вікна.

Ковзне середнє
ŷₜ = (1/T) Σ y_{t-k} (k від 0 до T-1)
Широке вікно → більш плавний тренд, але повільніша реакція на зміни. Вузьке вікно → чутливіше до змін, але залишається більше шуму. Для тижневих даних RetailPulse вікно T=4 (4 тижні) або T=12 (квартал) — типові стартові точки.
Ковзне середнє згладжує шум, виявляючи загальний тренд
Проблема ковзного середнього

Класичний МНК (мінімізація суми квадратів) мінімізує загальну помилку, але передбачає постійні параметри. Реальні часові ряди нестаціонарні: рівень продажів RetailPulse міняється через відкриття нових магазинів, сезони, кризи. Потрібні алгоритми, що «забувають» старе й адаптуються до нового.

Експоненціальне згладжування

Замість рівноваговит усіх T спостережень — зважуємо їх так, що новіші мають більшу вагу. Ваги спадають експоненціально із зростанням «старості» спостереження. Один параметр α ∈ (0,1) контролює швидкість «забування».

Просте експоненціальне згладжування (Brown)
ŷ_{t+1} = ŷₜ + α(yₜ − ŷₜ) [форма корекції стану]
ŷ_{t+1} = α·yₜ + (1−α)·ŷₜ [еквівалентна форма]

α → 1: ŷ ≈ yₜ (майже без згладжування)
α → 0: ŷ майже не змінюється (сильне згладжування)
Рекурентна форма «корекції помилки» (error correction) відкриває суть: нова оцінка = стара оцінка + α × (помилка прогнозу). Параметр α — це «швидкість навчання», яку підбирають мінімізацією суми квадратів помилок на навчальній вибірці.
Аналогія: термостат

Термостат корегує температуру пропорційно різниці між заданою і поточною температурою. Великий α — агресивне регулювання (реагує на кожне коливання). Малий α — плавне, інерційне регулювання. Задача — знайти баланс, де термостат стабільний, але реагує на справжні зміни.

Типова помилка: забути про «warm-up» SES

На першому кроці ŷ₁ = y₁ — модель "холодна". Перші 5–10 прогнозів нестабільні незалежно від α. При порівнянні SES з ARIMA на тестових даних завжди виключайте перші m (або 10) спостережень із розрахунку MAE/MAPE — інакше "warm-up ефект" завищить помилку SES.

Модель Холта — лінійний тренд

Просте згладжування добре справляється з трендом «постійний рівень», але не може правильно екстраполювати зростаючий або спадаючий тренд. Модель Холта (подвійне згладжування) явно оцінює і рівень, і нахил.

Модель Холта (Double Exponential Smoothing)
â_t = α·yₜ + (1−α)(â_{t-1} + b̂_{t-1}) [рівень]
b̂_t = β(â_t − â_{t-1}) + (1−β)b̂_{t-1} [нахил тренду]

Прогноз: ŷ_{t+d} = â_t + b̂_t·d
α — швидкість адаптації рівня, β — швидкість адаптації нахилу. Прогноз на d кроків вперед лінійно екстраполює поточний тренд. Якщо тренд стабільний — підходить чудово. Якщо тренд нелінійний або не постійний — потрібен «загасаючий тренд» (damped trend) або модель Холта–Вінтерса.

Модель Холта–Вінтерса — тренд + сезонність

Найпотужніший з класичних методів: три рівняння одночасно оновлюють рівень, нахил тренду і сезонний профіль. Є дві версії: адитивна (амплітуда сезонності стала) і мультиплікативна (амплітуда пропорційна до рівня).

Holt-Winters — адитивна версія
â_t = α(yₜ − ŝ_{t-m}) + (1−α)(â_{t-1} + b̂_{t-1}) [рівень]
b̂_t = β(â_t − â_{t-1}) + (1−β)b̂_{t-1} [нахил]
ŝ_t = γ(yₜ − â_t) + (1−γ)ŝ_{t-m} [сезонність]

Прогноз: ŷ_{t+d} = â_t + b̂_t·d + ŝ_{t+d-m}
m — довжина сезонного циклу (наприклад, m=52 для тижневих даних із річним циклом). Три параметри α, β, γ підбирають оптимізацією. Якщо амплітуда коливань зростає разом із трендом (що типово для RetailPulse — новорічний пік стає більшим у міру зростання мережі) — потрібна мультиплікативна версія.
Holt-Winters — мультиплікативна версія
â_t = α(yₜ / ŝ_{t-m}) + (1−α)(â_{t-1} + b̂_{t-1}) [рівень]
b̂_t = β(â_t − â_{t-1}) + (1−β)b̂_{t-1} [нахил]
ŝ_t = γ(yₜ / â_t) + (1−γ)ŝ_{t-m} [сезонність]

Прогноз: ŷ_{t+d} = (â_t + b̂_t·d) · ŝ_{t+d-m}
Ключова відмінність від адитивної: сезонність входить як множник, а не як доданок. Рівень ділиться на сезонний індекс (а не віднімає). Перевірка: якщо у адитивної моделі залишки мають зростаючу дисперсію — природа ряду мультиплікативна. Ознака для RetailPulse: якщо різдвяний пік у 2023 удвічі більший за 2020 — мультиплікативна.
Ініціалізація Holt-Winters (стандартний метод)
Потрібні перші m (мінімум 2m) спостережень для "warm-up": Крок 1 — Початковий рівень â₀: â₀ = (y₁ + y₂ + ... + yₘ) / m ← середнє першого сезону Крок 2 — Початковий нахил b̂₀: b̂₀ = [(yₘ₊₁+...+y₂ₘ) − (y₁+...+yₘ)] / m² (різниця між середніми двох перших сезонів) Крок 3 — Початкові сезонні індекси ŝ₋ₘ₊₁, ..., ŝ₀ (адитивна): ŝⱼ = yⱼ − â₀, j = 1, ..., m Нормуємо: ŝⱼ ← ŝⱼ − (1/m)Σŝⱼ (сума = 0) Для мультиплікативної: ŝⱼ = yⱼ / â₀ (середнє = 1)
Неправильна ініціалізація спричиняє нестабільний "розгін" на перших m–2m кроках. statsmodels.ExponentialSmoothing з initialization_method='estimated' оцінює початкові стани разом із α, β, γ методом MLE — це точніше, але потребує більших даних (N > 4m). Для RetailPulse (m=52): мінімум 104 тижні (~2 роки) для надійної ініціалізації. При менших вибірках — використовуйте initialization_method='heuristic'.
Log-перетворення: мультиплікативна → адитивна

Якщо yₜ = Tₜ · Sₜ · Rₜ, то log(yₜ) = log(Tₜ) + log(Sₜ) + log(Rₜ) — мультиплікативна модель стає адитивною після логарифмування. Практичний трюк: застосуйте np.log(y) → побудуйте адитивну Holt-Winters → прогнози поверніть через np.exp. Перевага: простіша інтерпретація, прогнози завжди додатні. Обмеження: не підходить, якщо ряд містить нулі або від'ємні значення.

Перетворення Бокса–Кокса

Логарифм — окремий випадок ширшого сімейства перетворень Бокса–Кокса, що стабілізують дисперсію і наближають розподіл до нормального. Параметр λ визначається з даних:

Перетворення Бокса–Кокса (Box–Cox)
y'ₜ = { (yₜλ − 1) / λ , λ ≠ 0 { log(yₜ) , λ = 0 Особливі випадки: λ = 1 → лінійне (без перетворення): y'ₜ = yₜ − 1 λ = 0 → натуральний логарифм: y'ₜ = ln(yₜ) λ = 0.5 → квадратний корінь: y'ₜ = √yₜ − 1 λ = −1 → зворотне: y'ₜ = 1/yₜ
Оптимальний λ: знаходять методом максимальної правдоподібності або профільним MLE — обирають λ, що мінімізує дисперсію залишків. В Python: scipy.stats.boxcox(y) повертає трансформовані дані і λ. Після прогнозу — обов'язково зворотне перетворення: scipy.special.inv_boxcox(y_hat, lambda_).
Не забудьте про зворотне перетворення!

Box-Cox перетворює ряд для навчання моделі. Прогнози будуються у трансформованому просторі, і їх обов'язково потрібно перетворити назад. Якщо цього не зробити, MAPE обчислюється на неправильному масштабі — завищена точність. Також пам'ятайте: після зворотного перетворення прогноз є медіаною, а не середнім — при λ=0 (log) середнє у вихідному просторі: exp(μ + σ²/2), де σ² — дисперсія залишків.

ETS-фреймворк — Error, Trend, Seasonality

ETS (Error–Trend–Seasonality) — це уніфікована формальна мова для опису всього сімейства методів експоненціального згладжування. Замість того, щоб пам'ятати різні формули для SES, Holt, Holt-Winters — кожна модель описується трьома літерами:

ETS-нотація: ETS(E, T, S)
E — Error (Помилка): A (Additive / адитивна) M (Multiplicative / мультиплікативна) T — Trend (Тренд): N (None / немає) A (Additive / лінійний) Ad (Additive damped / загасаючий) M (Multiplicative / мультиплікативний) S — Seasonality (Сезонність): N (None / немає) A (Additive / адитивна) M (Multiplicative / мультиплікативна) Приклади: ETS(A,N,N) = SES (просте згладжування Брауна) ETS(A,A,N) = Holt (подвійне згладжування) ETS(A,A,A) = Holt-Winters адитивний ETS(M,A,M) = Holt-Winters мультиплікативний ETS(A,Ad,N) = Damped trend (Гарднер–МакКензі)
Для кожної ETS-моделі існує стан-простірна (state-space) форма з оцінками максимальної правдоподібності. Це дозволяє: (1) об'єктивно порівнювати моделі за AIC/BIC; (2) автоматично генерувати інтервали прогнозування; (3) вибирати найкращу ETS-специфікацію через grid search. В Python: statsmodels.tsa.exponential_smoothing.ets.ETSModel.
ETS у RetailPulse: автоматичний вибір моделі

Для 50 000 SKUs ручний підбір моделі неможливий. ETSModel автоматично перебирає 24 специфікації та обирає ту, що мінімізує AICc. Для категорії «молочна продукція» з чіткою сезонністю і зростаючим трендом перемагає ETS(M,A,M) — Holt-Winters мультиплікативний. Для «бакалія без тренду» — ETS(A,N,A). Автоматизація скорочує час налаштування з тижнів до годин.

Поліноміальний тренд і теорема Брауна–Маєра

Метод Холта дає лінійну екстраполяцію — тренд зростає або спадає з постійною швидкістю. Але реальні тренди часто нелінійні: сповільнюються при насиченні ринку, прискорюються під час зростання. Природним розширенням є поліноміальний тренд.

Поліноміальний тренд порядку d
Tₜ = a₀ + a₁t + a₂t² + ... + aₐtᵈ

d=1: лінійний (Holt) → постійна швидкість зміни
d=2: квадратичний → прискорення / уповільнення
d=3: кубічний → S-подібні криві зростання
Параметри оцінюють МНК або методом двостороннього ковзного середнього. Але поліноміальні тренди небезпечні за межами горизонту: поліном будь-якого степеня прямує до ±∞. Для прогнозування на 1–2 сезони вперед краще використовувати загасаючі (damped) моделі.
Теорема Брауна–Маєра

Ця теорема доводить, що метод експоненціального згладжування для поліноміального тренду порядку d еквівалентний МНК-оцінці поліному ступеня d на зваженій вибірці, де ваги спадають геометрично в минуле. Практичний наслідок: DES (Holt) оптимально підходить для лінійного тренду так само, як SES — для постійного. Для квадратичного тренду аналогічно використовують потрійне згладжування або методи вищого порядку.

Загасаючий тренд (Damped Trend)

Стандартний метод Холта екстраполює тренд нескінченно: якщо продажі зараз зростають на 5% на тиждень, модель прогнозує вічне зростання. На довгих горизонтах це нереалістично — реальні тренди загасають.

Загасаючий тренд Гарднера–Маккензі (Damped Holt)
Рівень: Lₜ = α·yₜ + (1−α)·(Lₜ₋₁ + φ·Bₜ₋₁)
Тренд: Bₜ = β·(Lₜ − Lₜ₋₁) + (1−β)·φ·Bₜ₋₁

Прогноз на d кроків: ŷₜ₊ₐ = Lₜ + (φ + φ² + ... + φᵈ)·Bₜ

φ ∈ (0,1) — коефіцієнт загасання тренду
При φ=1 — стандартний Holt (лінійний тренд назавжди). При φ→0 — SES (тренд миттєво зникає). Типові значення: φ = 0.80–0.98. Вибирають оптимізацією по MSE на навчальній вибірці. Для RetailPulse φ≈0.90 часто дає кращий результат на 4+ тижневому горизонті.

Q&A
Питання для самоперевірки
01Чим відрізняється ковзне середнє від експоненціального згладжування?
+
Ковзне середнє дає рівні ваги T останнім спостереженням і нуль — всім попереднім (різкий «обрив»). Експоненціальне згладжування дає ваги, що спадають геометрично: найновіше спостереження — вага α, попереднє — α(1−α), позапопереднє — α(1−α)², і так далі до нескінченності (але практично дуже малі). ЕЗ «пам'ятає» всю історію, але з поступовим загасанням. Також ЕЗ природно записується у рекурентній формі й може прогнозувати вперед.
02Що станеться з прогнозом Holt, якщо β → 0?
+
При β → 0 нахил тренду b̂_t майже не оновлюється — модель «замерзає» у початковому нахилі. Це корисно, якщо тренд дуже стабільний і ми не хочемо, щоб тимчасові коливання його змінили. При β → 1 нахил оновлюється дуже агресивно, майже лінійно прив'язуючись до останньої різниці. Практично β ≈ 0.1–0.3 зазвичай дає кращі результати.
03Коли потрібна мультиплікативна версія Holt-Winters замість адитивної?
+
Мультиплікативна потрібна, коли амплітуда сезонних коливань пропорційна поточному рівню ряду. Ознака: на графіку сезонні «гірки» стають вищими у міру зростання тренду. Для RetailPulse: якщо у 2022 р. новорічний пік давав +200 тис. грн, а у 2025 р. (після зростання мережі) — +800 тис., то сезонність мультиплікативна. Якщо б пік залишився +200 тис. незалежно від загального обсягу — адитивна.
04Теорема Брауна–Маєра — що вона стверджує і навіщо потрібна?
+
Теорема Брауна–Маєра встановлює лінійний зв'язок між показниками вищого порядку експоненціальних середніх (ŷ[1], ŷ[2], ..., ŷ[n+1]) і коефіцієнтами полінома тренду (c₀, c₁, ..., cₙ). Це означає: замість того, щоб розв'язувати систему рівнянь для поліноміального тренду кожного разу з нуля, можна просто обчислити n+1 середніх і отримати коефіцієнти через одне матричне множення. Практична цінність: рекурентно оновлювати поліноміальний тренд складності n у режимі реального часу без матричних обернень.

Циклічні компоненти та SSA

Хвилі ділового циклу, багаторічні тренди, нерегулярні коливання попиту — ці явища існують у даних, але їх неможливо виявити через Фур'є-аналіз або Holt-Winters, бо вони не мають фіксованого відомого заздалегідь periodу. Метод сингулярного спектрального аналізу (SSA) вирішує цю задачу без жодних апріорних припущень про структуру сигналу.

Чому Фур'є-аналіз не підходить для реальних рядів

Перетворення Фур'є є потужним інструментом для аналізу стаціонарних, нескінченних сигналів. Але реальні часові ряди — короткі, зашумлені, нестаціонарні. Застосування Фур'є до них породжує спектральні артефакти: хибні частоти, яких у сигналі немає, і відсутність справжніх, які не помітні через шум.

Аналогія: зіпсований мікрофон

Уявіть, що ви записуєте мелодію зіпсованим мікрофоном, що додає шум і спотворює сигнал, — і потім намагаєтеся знайти ноти через спектрограму. Замість справжньої мелодії ви побачите хмару шуму й фальшиві «призраки» нот. SSA — це алгоритм, що відокремлює справжню мелодію від шуму навіть із поганого запису.

Ідея SSA: від ряду до матриці й назад

SSA (Singular Spectrum Analysis, або метод «Гусениця») ґрунтується на ключовому прийомі — перетворити одновимірний часовий ряд у багатовимірну матрицю («траєкторну матрицю»), а потім розкласти її через SVD. Метод не вимагає жодних апріорних припущень про тип компонент.

SSA pipeline: від вихідного ряду до виокремлених компонент

Крок 1–2: Вкладення та траєкторна матриця

Вибираємо довжину вікна L (параметр методу) і формуємо матрицю X розміром L × K, де K = N − L + 1. Кожен стовпець — це «зсунуте вікно» довжиною L:

Траєкторна (Ганкелева) матриця
X = [Y₁ | Y₂ | ... | Yₖ]
де Yᵢ = (yᵢ, yᵢ₊₁, ..., yᵢ₊ₗ₋₁)ᵀ — вектор-вікно

Властивість: X є симетричною відносно антидіагоналі (матриця Ганкеля)
Вибір L критично важливий: L має бути достатньо великим, щоб охопити один цикл (L ≥ m, де m — очікуваний період), але не більше N/2. Для річних даних із тижневою частотою (N=200) і 52-тижневим циклом — L ≈ 52–100.

Власні числа і вектори — фундамент SVD

SVD спирається на апарат власних чисел і векторів. Перш ніж перейти до сингулярного розкладу, розберемо ці поняття геометрично — без них SVD лишається «чорним ящиком».

Власне число і власний вектор
A · v = λ · v

v ≠ 0 — власний вектор (eigenvector) матриці A
λ — власне число (eigenvalue), що відповідає v

Характеристичне рівняння: det(A − λI) = 0
Геометрично: власний вектор — це напрям, вздовж якого лінійне перетворення A лише «масштабує» вектор, не повертає його. λ — коефіцієнт масштабування. Від'ємне λ означає відображення. Симетрична матриця завжди має речові власні числа та ортогональні власні вектори.
Аналогія: власні моди коливань

Уявіть пружну систему: якщо вдарити довільно, вона коливатиметься хаотично. Але є власні напрями (моди коливань), вдаривши вздовж яких, система коливається рівно в одному напрямку — без перехресних рухів. Це і є власні вектори. Власне число — «енергія» (частота) коливань у цьому напрямку. SSA використовує той самий принцип: знаходить власні напрями варіації часового ряду через матрицю XᵀX. Найбільше власне число відповідає тренду (найпотужнішій структурі), менші — циклам і шуму.

Власний вектор: перетворення A лише масштабує напрям, не повертає його

У контексті SSA матриця XᵀX є квадратною симетричною — її власні вектори завжди ортогональні й утворюють нову «систему координат» у просторі вікон. Сингулярні вектори uᵢ, vᵢ у SVD — це власні вектори матриць XXᵀ і XᵀX відповідно, а сингулярні значення σᵢ = √λᵢ.

Крок 3: Сингулярний розклад (SVD)

Ганкелеву матрицю X розкладаємо через SVD: X = UΣVᵀ. Отримуємо r = rank(X) «елементарних» матриць, упорядкованих за спаданням сингулярних значень σ₁ ≥ σ₂ ≥ ... ≥ σᵣ.

SVD декомпозиція
X = Σᵢ σᵢ · uᵢ · vᵢᵀ = X₁ + X₂ + ... + Xᵣ

σᵢ² — власні числа матриці XᵀX
uᵢ — ліві сингулярні вектори (власні вектори XXᵀ)
vᵢ — праві сингулярні вектори (власні вектори XᵀX)
Кожна складова Xᵢ = σᵢuᵢvᵢᵀ — це «елементарна матриця», що відповідає одній моді коливань. Великі σᵢ → домінантні компоненти (зазвичай тренд або сильні гармоніки). Малі σᵢ → шум.

Крок 4–5: Групування та відновлення

Групуємо елементарні матриці у групи — тренд, гармоніки, шум — і для кожної групи відновлюємо одновимірний ряд через діагональне усереднення (ганкелізацію). Ключ до успіху — правильне групування, яке роблять на основі:

  • Внесок σᵢ у загальну дисперсію (перші вектори — тренд)
  • Парне розміщення на «діаграмі розсіювання» парних головних компонент (ознака гармоніки)
  • Візуальний аналіз реконструйованих компонент
Scree plot: перші 5 компонент несуть сигнал (тренд + гармоніки), решта — шум
Аналогія: музичний розклад

Оркестр грає симфонію — це і є ваш часовий ряд. SVD «розкладає» запис на окремі інструменти: перша скрипка (тренд, найгучніший), далі секції оркестру (гармоніки з різними частотами), наприкінці — шерех зали (шум). Після «ізоляції» кожного інструменту їх можна прогнозувати незалежно й потім скласти назад.

SSA vs Фур'є: ключова відмінність

SSA — модельно-незалежний: він не припускає, що сигнал є сумою синусів. Він лише знаходить домінуючі структури в даних. Тому SSA ефективно виявляє аперіодичні тренди, «наполовину-циклічні» коливання ділового циклу й навіть поступові зміни у структурі ряду — все те, з чим Фур'є безсилий. Але SSA потребує більше даних: загальне правило — N ≥ 4L, де L ≥ 2m.

SSA-прогнозування — рекурентний алгоритм (r-SSA)

SSA-декомпозиція дає минуле — як же зробити прогноз? Рекурентний SSA (r-SSA) використовує власні вектори U для побудови лінійного рекурентного співвідношення (ЛРС), що продовжує кожну компоненту вперед:

Рекурентне SSA-прогнозування (r-SSA)
Крок 1. Знаходимо «прогнозний вектор» R: R = (1/(1 − π²)) · Σᵢ∈I πᵢ·Uᵢ[1:L-1] де Uᵢ — обрані власні вектори (з групи I — сигнал) πᵢ = Uᵢ[L] — останній елемент i-го власного вектора π² = Σᵢ∈I πᵢ² (має бути < 1; інакше ЛРС нестійке) Крок 2. Лінійне рекурентне співвідношення: ŷₙ₊ₕ = Σⱼ₌₁ᴸ⁻¹ Rⱼ · ŷₙ₊ₕ₋ⱼ (прогнозований хʼ₋ᵢ — елементи реконструйованого сигналу) Крок 3. Повторюємо h разів для горизонту h.
Умова стійкості: π² < 1. Якщо π² близьке до 1 (малий знаменник), прогноз нестабільний. Це аналог умови стаціонарності для ARIMA. Для тренду π² може бути > 0.99 — тренд продовжується, але шум наростає швидко. На практиці: прогнозуйте тренд і кожну гармоніку окремо, потім складайте. Q&A
Питання для самоперевірки
01Чому SSA перевершує Фур'є для реальних зашумлених нестаціонарних рядів?
+
Фур'є оптимальний для стаціонарних нескінченних сигналів, що є сумою синусоїд. Реальні ряди — короткі, нестаціонарні, зашумлені, і їхні «цикли» не мають точного відомого заздалегідь periodу. Фур'є на таких даних породжує артефакти — хибні частоти через кінцевість вибірки і шум. SSA не припускає ніякої форми сигналу: він просто знаходить структури через SVD. Погані сингулярні числа (шум) автоматично відкидаються при групуванні.
02Як вибрати довжину вікна L в SSA?
+
L повинна бути: (1) достатньо великою, щоб охопити найдовший очікуваний цикл (L ≥ m), (2) значно меншою за N (L ≤ N/2, краще N/3). Якщо відома сезонність — L = m або L = 2m. Якщо невідома — перевіряють кілька значень і порівнюють якість реконструкції. Для RetailPulse (тижневі дані, 3 роки = N=156): L=52 (річний цикл) або L=26 (напівріч) є розумними стартовими точками.
03Як у SSA відрізнити тренд від гармоніки, а гармоніку від шуму?
+
Тренд: перший (або перший-другий) сингулярний вектор, монотонний, поступово змінюється, відповідає найбільшому σ₁. Гармоніка: пара сингулярних векторів зі схожими σᵢ, що утворюють окружність на «діаграмі розсіювання» (scatter plot pᵢ vs pⱼ). Візуально реконструйована компонента виглядає як синусоїда. Шум: малі σᵢ, реконструйовані компоненти нагадують білий шум (немає структури в ACF). На практиці 80% внеску перших K компонент — хороший орієнтир для відсікання шуму.

Траєкторна матриця L×K, SVD, діагональне усереднення компонент, scree plot внесків
import numpy as np

def diagonal_average(M):
    """Ганкелізація: матриця → ряд діагональним усередненням"""
    L, K = M.shape
    N = L + K - 1
    result = np.zeros(N)
    for k in range(N):
        diag = [M[k - j, j] for j in range(max(0, k - K + 1),
                                             min(L, k + 1))]
        result[k] = np.mean(diag)
    return result

def ssa(series, L):
    """SSA: embedding → SVD → компоненти → діагональне усереднення"""
    N = len(series)
    K = N - L + 1

    # Крок 1-2: Траєкторна матриця X (L × K)
    X = np.array([series[i:i + L] for i in range(K)]).T

    # Крок 3: SVD
    U, sigma, Vt = np.linalg.svd(X, full_matrices=False)

    # Крок 4: Окремі елементарні матриці та їх усереднення
    components = []
    for i in range(len(sigma)):
        Xi = sigma[i] * np.outer(U[:, i], Vt[i, :])
        components.append(diagonal_average(Xi))

    return components, sigma, U, Vt

Коефіцієнти ЛРС з власних векторів U; рекурентний прогноз на steps кроків
import numpy as np

def ssa_forecast(series, U, r, steps):
    """r-SSA прогнозування через лінійне рекурентне співвідношення"""
    L = U.shape[0]
    Ur = U[:, :r]                              # перші r власних векторів
    nu2 = np.sum(Ur[-1, :] ** 2)              # вертикальність
    a   = (Ur[:-1, :] @ Ur[-1, :]) / (1 - nu2)  # коефіцієнти ЛРС

    buf = list(series[-(L - 1):])             # останні L-1 значень
    for _ in range(steps):
        next_val = float(np.dot(a, buf[-(L - 1):]))
        buf.append(next_val)
    return np.array(buf[L - 1:])

# Прогноз на 13 тижнів (r=6)

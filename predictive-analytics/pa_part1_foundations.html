Вступ до предиктивної аналітики

Предиктивна аналітика — це не просто прогнозування майбутнього. Це систематичний процес: від розуміння бізнес-задачі до розгортання моделі у production. Розбираємо, чим вона відрізняється від описового та прескриптивного аналізу, чому невизначеність — невід'ємна частина гри, і як CRISP-DM допомагає не загубитись на шляху від сирих даних до рішення.

Три рівні аналізу даних

Будь-який аналіз даних можна розмістити на шкалі від «що вже сталося» до «що з цим робити». Ці три рівні ієрархічні: кожен наступний будується на попередньому, але дає більше цінності й вимагає складніших інструментів.

Аналогія

Уявіть лікаря. Описовий аналіз — це результати аналізів та огляд («у пацієнта температура 38.5°C, тиск 140/90»). Предиктивний — прогноз перебігу хвороби («з ймовірністю 70% стан погіршиться без лікування»). Прескриптивний — рекомендація дії («призначити антибіотик X, курс 7 днів, контроль через 3 дні»).

D
Описовий аналіз

Що відбулося? Підсумовує минуле через звіти, графіки, середні значення. Відповідає на питання «де ми зараз». Приклад: продажі за квартал по регіонах.

P
Предиктивний аналіз

Що може статися? Виявляє закономірності в минулому і проектує їх у майбутнє. Статистичні моделі, часові ряди, ML. Приклад: прогноз продажів на наступний місяць.

Pr
Прескриптивний аналіз

Що потрібно зробити? Рекомендує дію на основі прогнозу. Оптимізація, симуляція сценаріїв. Приклад: оптимальний обсяг замовлення товару в кожному магазині.

Стадії процесу предиктивної аналітики: CRISP-DM

CRISP-DM (Cross-Industry Standard Process for Data Mining) — найпопулярніша методологія для структурування проектів з аналізу даних. Процес є ітеративним: реальні проекти рідко проходять стадії лінійно — частіше ви повертаєтеся назад після нових знахідок.

Модель аналітичної зрілості Gartner

Gartner описує еволюцію аналітики як чотири рівні зростаючої складності та цінності. Кожен наступний рівень дає більше відповідей — але вимагає більше даних, інструментів і компетенцій.

Gartner Analytics Ascendancy Model — чотири рівні аналітичної зрілості

Цей курс зосереджений на предиктивному рівні: ми навчаємося будувати моделі, що проектують минулі закономірності у майбутнє. Прескриптивна аналітика (оптимізація запасів на основі прогнозу RetailPulse) — природне продовження.

Невизначеність у предиктивній аналітиці

Прогноз — це не точне передбачення. Це оцінка найбільш імовірного майбутнього з кількісно визначеною мірою невпевненості. Ігнорування невизначеності — одна з найпоширеніших помилок, яка перетворює прогноз на ілюзію точності.

А
Алеаторична

Невідновлювана, властива системі. Випадковий шум у даних: непередбачувана поведінка клієнтів, погода, збої. Не зменшується з кількістю даних.

Е
Епістемічна

Через неповноту знань. Зменшується з накопиченням даних і покращенням моделі. Наприклад, погана точність через малу вибірку або неправильну специфікацію моделі.

CI
Довірчий інтервал

Практичний спосіб комунікувати невизначеність. Замість «продажі будуть 1000 одиниць» — «з ймовірністю 95% від 850 до 1160 одиниць».

Типова помилка

Точковий прогноз без інтервалу невизначеності — це як прогноз погоди «завтра 22°C» без позначки «±5°C». Бізнес-рішення, що ґрунтуються на таких прогнозах, систематично недооцінюють ризик. Завжди вказуйте інтервал прогнозування разом із точковою оцінкою.

Описова та індуктивна статистика

Перш ніж будувати предиктивні моделі, потрібно описати дані — зрозуміти, з чим маєш справу. Описова статистика відповідає на питання «що є у вибірці», тоді як індуктивна (inferential) — «що можна вивести про генеральну сукупність на основі вибірки».

Три кити описової статистики
Центральна тенденція: середнє x̄ = (1/n) Σ xᵢ, медіана, мода
Розсіювання: дисперсія σ² = (1/n) Σ(xᵢ − x̄)², ст. відхилення σ
Форма розподілу: асиметрія (skewness), ексцес (kurtosis)
Дисперсія вимірює середнє квадратичне відхилення від центру. Чому квадрат? Щоб від'ємні відхилення не компенсували додатні. Стандартне відхилення — це дисперсія, повернена до одиниць вимірювання.

Шкали вимірювання

Не всі числа є однаковими. Тип шкали визначає, які статистичні операції є коректними. Плутанина зі шкалами — джерело великої кількості невірних висновків.

Аналогія: абсолютний нуль

Температура 0°C не означає «відсутність тепла» — це просто точка замерзання води (умовна). Тому не можна сказати «20°C удвічі тепліше за 10°C». Але 0 гривень продажів — це справді «нуль продажів», тому відношення мають сенс: 20 000 грн — це вдвічі більше за 10 000 грн.

Метод максимальної правдоподібності (MLE)

Більшість статистичних моделей потребують оцінки параметрів за даними. MLE — найпоширеніший підхід: знайти параметри, за яких спостережені дані є найбільш імовірними.

MLE — ключова ідея
L(θ | data) = P(data | θ) → max

θ̂_MLE = argmax_θ Σ log P(xᵢ | θ)
Функцію правдоподібності максимізують у логарифмічній формі (log-likelihood) — це перетворює добуток ймовірностей на суму логарифмів, що числово набагато зручніше. Для нормального розподілу MLE дає ті самі оцінки, що й метод найменших квадратів.
Аналогія: монета та MLE

Ви підкинули монету 10 разів і отримали 7 орлів. MLE-оцінка ймовірності орла: 7/10 = 0.7. Це саме те значення, за якого спостережена послідовність (7 орлів з 10) є найбільш ймовірною. Простіша задача — але те саме математичне мислення використовується для оцінки параметрів ARIMA-моделі з тисячами точок.

Зміщення та дисперсія — декомпозиція MSE

Розуміти, звідки береться помилка моделі — так само важливо, як уміти її вимірювати. Будь-яку помилку прогнозу можна розкласти на три незалежні джерела.

Декомпозиція середньоквадратичної помилки
MSE = Bias² + Variance + Noise

Bias² = (E[ŷ] − y*)² — систематична помилка моделі
Variance = E[(ŷ − E[ŷ])²] — чутливість до навчальних даних
Noise = σ² — невідновлювана алеаторична помилка
y* — справжнє значення, ŷ — прогноз моделі. Noise не залежить від моделі — це обмеження самих даних. Bias і Variance — наслідки вибору моделі, і між ними існує фундаментальний компроміс: зменшуючи одне, ми збільшуємо інше.
Bias-Variance Tradeoff: проста модель — велике Bias², складна — велике Variance
Bias
Зміщення (Underfitting)

Модель занадто проста, не захоплює реальних закономірностей. Наприклад, лінійний тренд на явно нелінійному ряді. Помилка систематична і стала на будь-яких даних.

Var
Дисперсія (Overfitting)

Модель занадто складна — «вивчила» шум навчальної вибірки. На нових даних помилка різко зростає. ARIMA(5,1,5) на 50 точках — типовий приклад.

Opt
Оптимальна складність

Мінімум Total MSE — баланс між Bias² і Variance. Знаходять через backtest на часовому ряді або критерії AIC/BIC (штраф за кількість параметрів).

Q&A
Питання для самоперевірки
01Чим предиктивний аналіз принципово відрізняється від описового?
+
Описовий відповідає на питання «що вже сталося» і базується на фактах — звіти, агрегати, підсумки. Предиктивний відповідає на «що може статися» і будує модель, яка проектує виявлені закономірності у майбутнє. Ключова відмінність: описовий має справу з відомими даними, предиктивний — із оцінкою невідомого майбутнього, завжди з певним рівнем невизначеності.
02Які стадії CRISP-DM і чому процес ітеративний?
+
Шість стадій: Business Understanding → Data Understanding → Data Preparation → Modeling → Evaluation → Deployment. Ітеративний, бо реальність не вкладається в одноразовий прохід: після оцінки моделі може з'ясуватися, що задача сформульована неправильно (повернення до Business Understanding), або що потрібні нові ознаки (повернення до Data Preparation). CRISP-DM — це не водоспад, а цикл.
03Коли MLE дає ті самі результати, що й МНК?
+
Якщо припускати, що залишки моделі розподілені нормально з постійною дисперсією, то максимізація функції правдоподібності еквівалентна мінімізації суми квадратів залишків (МНК). Тому для лінійної регресії з нормальними залишками обидва підходи дають однаковий результат. Але MLE ширший: він застосовується для будь-якого розподілу (пуасонівського, біноміального тощо), а не лише нормального.
04Чому важливо вказувати інтервал невизначеності разом із прогнозом?
+
Точковий прогноз — це завжди оцінка з помилкою. Без інтервалу невизначеності бізнес не бачить ризику і приймає рішення так, ніби прогноз точний. Наприклад, якщо RetailPulse прогнозує «продамо 1000 одиниць», але реальний інтервал — 600–1400, то стратегія замовлення буде принципово різною. Інтервал дозволяє оцінити ризик і прийняти рішення з урахуванням вартості помилки.

import numpy as np
import matplotlib.pyplot as plt

# Завантаження даних RetailPulse
df = pd.read_csv('retail_sales.csv', parse_dates=['date'])

# Тижнева агрегація одного SKU
weekly = (df[df['sku_id'] == 'MILK-1L-007']
          .resample('W')['sales'].sum())

# Описова статистика
print(weekly.describe())

print(f"Коефіцієнт варіації: {cv:.2%}")

print(f"Ексцес:              {weekly.kurtosis():.3f}")

# Гістограма + boxplot

Часові ряди — визначення та декомпозиція

Часовий ряд — це послідовність спостережень, впорядкованих у часі. На перший погляд просто. Але саме ця «впорядкованість у часі» перевертає всю класичну статистику догори ногами: значення не незалежні, минуле впливає на майбутнє, і це не баг, а головна ознака, яку ми використовуємо для прогнозування.

Прогноз vs прогнозування — тонка відмінність

Прогнозування (forecasting) — передбачення майбутніх значень однієї і тієї ж змінної на основі її минулих значень. Тут роль відіграє час. Предикція (prediction) — ширше поняття: оцінка залежної змінної за значеннями незалежних, не обов'язково пов'язана з часом.

Аналогія

Прогнозування погоди — «forecasting»: завтрашня температура прогнозується на основі минулих температур і атмосферних даних. Оцінка ціни квартири за її площею і районом — «prediction» (регресія): тут час не є ключовим фактором. У курсі ми зосереджуємось саме на forecasting.

Часовий ряд — формальне визначення

Часовий ряд — впорядкована послідовність значень y₀, y₁, y₂, ..., yₙ, де індекс відповідає рівномірно розподіленим моментам часу. Три ключові особливості відрізняють часові ряди від звичайних вибірок:

  • Впорядкованість: порядок спостережень принципово важливий — перемішавши значення, ми знищимо весь корисний сигнал
  • Залежність між сусідніми значеннями: продажі сьогодні залежать від продажів вчора, а ті — від позавчорашніх
  • Невідома модель: на відміну від фізики, ми рідко знаємо «справжній» закон генерації даних — доводиться підбирати модель
Схематичний часовий ряд продажів RetailPulse — тренд + сезонність

Пропущені значення — методи відновлення

Реальні часові ряди рідко бувають ідеально повними. Збої обладнання, помилки збору даних, вихідні у магазинах — все це створює пропуски. Правильний підхід залежить від механізму пропуску і частки відсутніх спостережень:

Видалення (Listwise deletion)

Просто прибрати рядки з пропуском. Безпечно лише якщо пропуски MCAR (Missing Completely At Random) і їх < 5%. Для часових рядів майже ніколи не підходить — порушує часову структуру.

Лінійна інтерполяція

Заповнення пропуску прямою між сусідніми відомими точками. Добре для коротких одиничних пропусків у гладких рядах. Погано для різких змін і сезонних рядів — «розмиває» пік.

Сезонна імпутація

Замінити пропуск значенням з того ж сезону попереднього (або кількох попередніх) років. Для RetailPulse: якщо немає продажів у тиждень 3 цього року — беремо середнє тижня 3 за 2021–2023. Зберігає сезонну структуру.

Поширені методи імпутації
Forward Fill (ffill): yₜ = yₜ₋₁ (взяти попереднє відоме значення) Backward Fill (bfill): yₜ = yₜ₊₁ (взяти наступне відоме) Лінійна інтерпол.: yₜ = yₐ + (t−a)/(b−a) · (y_b − yₐ) Сезонна медіана: yₜ = median(yₜ₋ₘ, yₜ₋₂ₘ, ...) (m — сезонний лаг) LOCF + Holt-Winters: Навчити модель на наявних даних → передбачити пропуски
Правило вибору: (1) 1 пропуск → ffill або лінійна; (2) 2–4 пропуски → сезонна медіана; (3) >1 тижня пропусків → Holt-Winters або KNN-регресор з часовими ознаками. Для RetailPulse pропуски >4 тижнів позначаються прапорцем «no_data» і виключаються із навчальної вибірки.

Викиди — виявлення та обробка

Викид (outlier) — спостереження, що різко відрізняється від загального патерну ряду. Для прогнозування вони небезпечні: один великий викид може значно зсунути оцінки параметрів моделі. Але не всі викиди — помилки! Бум продажів через акцію — реальна подія, яку потрібно пояснити, а не видалити.

RetailPulse: 3 типи «викидів»

(1) Помилка збору: магазин передав нуль продажів через збій касового ПЗ — насправді продавали. Рішення: імпутація сезонною медіаною. (2) Акційний пік: продажі 500% від норми через промо-акцію. Рішення: зберегти значення, додати dummy-ознаку «є_акція» у модель. (3) Форс-мажор: нульові продажі через закриття магазину на ремонт — відома причина. Рішення: виключити з навчання і прогнозування. Правило: завжди з'ясовуй причину перед видаленням.

Викид у навчальних даних ≠ проблема у прогнозі

Якщо викид — реальна подія (акція, свято, криза), що може повторитись, — видалення його з навчальних даних погіршить прогноз майбутніх подібних подій. Краще додати пояснювальну змінну (dummy або кількісну ознаку «величина акційної знижки»). Видалення виправдане лише для справжніх помилок вимірювання.

Теорема Найквіста–Шеннона (дискретизація)

Часовий ряд завжди є дискретним записом безперервного процесу. Виникає фундаментальне питання: як часто треба вимірювати, щоб не втратити інформацію?

Теорема Найквіста–Шеннона
fₛ ≥ 2 · f_max

fₛ — частота дискретизації (вимірювань на одиницю часу)
f_max — максимальна частота в сигналі

Або: Δt ≤ 1 / (2 · f_max) ← мінімальний крок між вимірами
Якщо процес має сезонний цикл 7 днів (тижнева сезонність), f_max = 1/7 циклів/день — і для коректного відтворення потрібно вимірювати частіше ніж раз на 3.5 дні. Щоденні дані (Δt=1 день) — достатні. Якщо вимірювати лише раз на тиждень — тижнева сезонність повністю «зникне» у даних (ефект аліасингу).
Аналогія: аліасинг у роздрібній торгівлі

Уявіть, що RetailPulse збирає дані лише раз на 2 тижні. Тижнева сезонність (п'ятниця = пік продажів) стане невидимою або деформується — ефект аліасингу. Ряд виглядатиме «гладкішим», але інформацію про реальний пік буде безповоротно втрачено. Практичний висновок: частота збору даних має відповідати горизонту та циклічності прогнозу.

Спектральний аналіз — частотний погляд на ряд

Часовий ряд можна розглядати у двох рівноцінних площинах: у часовій (значення yₜ від t) і в частотній (яка «потужність» є у кожній частоті). Спектральна щільність — це розподіл дисперсії ряду по частотах:

Спектральна щільність і Periodogram
Спектральна щільність: f(ω) = (1/2π) · Σₖ₌₋∞^∞ γ(k) · e^{−iωk}, ω ∈ [−π, π] γ(k) — автоковаріація при лагу k ω — частота (радіани/крок), ω = 2π/T для циклу T кроків Інтерпретація: ∫₋π^π f(ω) dω = Var[yₜ] (площа під спектром = загальна дисперсія) Вибірковий periodogram (оцінка f): I(ωⱼ) = (1/2πn) · |Σₜ yₜ · e^{−iωⱼt}|², ωⱼ = 2πj/n
Пік у periodogram на частоті ω = 2π/52 означає річну сезонність (цикл 52 тижні). Пік на ω = 2π/7 — тижнева сезонність (цикл 7 днів). Periodogram дозволяє виявити сезонність без апріорних припущень. В Python: scipy.signal.periodogram(y, fs=1). Обмеження: сирий periodogram — шумна оцінка; для реальних даних використовують згладжений (Welch-метод) або SSA.
Аналогія: спектр = «ЕКГ» ряду

ЕКГ показує серцевий ритм у часі. Спектрограма кардіолога показує, які частоти домінують у сигналі (основний ритм, обертони). Для часового ряду продажів RetailPulse: periodogram покаже три чіткі піки — тижневий (ω=2π/7), квартальний (2π/13) і річний (2π/52). Це підказує параметр m для SARIMA і кількість гармонік для SSA — ще до будь-якого моделювання.

Багатовимірні часові ряди

Реальні системи рідко описуються однією змінною. Багатовимірний часовий ряд — вектор Yₜ = (y₁ₜ, y₂ₜ, ..., yₘₜ)ᵀ, де кожна компонента — окремий ряд. Між компонентами існують складні взаємні залежності.

При m=10, p=2: вже 210 параметрів — кожен потребує даних для оцінки. Без жорсткої регуляризації або відбору змінних модель переоснащується. Крос-кореляційна функція (CCF) — пошук lead-lag зв'язків

АКФ вимірює кореляцію ряду із самим собою. CCF (Cross-Correlation Function) — кореляцію між двома різними рядами при різних зсувах. Це головний інструмент для виявлення того, чи передує один ряд іншому (lead-lag relationship):

Крос-кореляційна функція CCF(k)
CCF(k) = Cor(xₜ, yₜ₊ₖ) = γ_xy(k) / √(γ_x(0)·γ_y(0)) k > 0: xₜ корелює з yₜ₊ₖ → x випереджає y на k кроків (x = leading indicator) k < 0: xₜ корелює з yₜ₋|k| → y випереджає x на |k| кроків k = 0: синхронна кореляція CCF(k) ∈ [−1, 1] Значуща при |CCF(k)| > 1.96/√n (95%-рівень)
Важливо: перед обчисленням CCF обидва ряди мають бути стаціонарними — інакше висока CCF може бути хибною (spurious correlation через спільний тренд). Диференціюйте або видаляйте тренд перед аналізом. В Python: statsmodels.tsa.stattools.ccf(x, y).
RetailPulse: CCF ціни та попиту

Відділ ціноутворення знижує ціну → попит зростає, але з затримкою. Обчислюємо CCF між Δprice_t та Δdemand_t. Якщо CCF має пік при k=2 (тижні), це означає: зниження ціни сьогодні дасть ефект попиту за 2 тижні. Це виявлений lead-lag: ціна — leading indicator для попиту. Практичне застосування: включаємо price_{t−2} як екзогенну змінну в ARIMAX.

Декомпозиція часового ряду

Класична ідея: будь-який часовий ряд можна представити як суму (або добуток) кількох компонент. Це спрощення, але воно надзвичайно корисне на практиці — дозволяє аналізувати та прогнозувати кожну компоненту окремо.

Адитивна та мультиплікативна моделі
Адитивна: yₜ = Tₜ + Sₜ + Cₜ + Rₜ
Мультиплікативна: yₜ = Tₜ × Sₜ × Cₜ × Rₜ
T — тренд, S — сезонність, C — циклічна компонента, R — залишки (шум). Адитивну модель використовують, коли амплітуда сезонних коливань стала. Мультиплікативну — коли вона зростає разом із трендом (тобто «25% від поточного рівня», а не «завжди +100 одиниць»).
Сезонно скоригований ряд (Seasonally Adjusted)
Адитивна: SA_t = yₜ − Ŝₜ = Tₜ + Cₜ + Rₜ Мультиплікативна: SA_t = yₜ / Ŝₜ = Tₜ × Cₜ × Rₜ де Ŝₜ — оцінка сезонної компоненти (з STL, Census X-13 або класичної декомпозиції)
SA-ряд використовується: (1) для аналізу тренду без «сезонного шуму» — наприклад, дані Держстату про ВВП і безробіття завжди подаються у SA-формі; (2) як вхід для ARIMA-моделювання (якщо D=0 у SARIMA); (3) для порівняння між різними сезонами. Інструменти: statsmodels STL → result.seasonal, або seasonal_decompose(y, model='additive').seasonal. Для RetailPulse SA-ряд дозволяє виявити справжній тренд продажів незалежно від свят і сезону.
T
Тренд (Trend)

Довгострокова тенденція до зростання або спадання. Плавна, без різких коливань. Приклад RetailPulse: загальне зростання продажів через відкриття нових магазинів.

S
Сезонність (Seasonality)

Регулярні коливання з фіксованим відомим періодом. Тижневий (пік у вихідні), місячний (аванс/зарплата), річний (Новий рік, Великдень) — це все сезонність.

C
Циклічний компонент

Коливання без фіксованого відомого заздалегідь періоду. Бізнес-цикли, кризи. Відрізняється від сезонності тим, що тривалість циклу заздалегідь невідома.

R
Залишки (Residuals)

Те, що залишилося після видалення T, S, C. В ідеалі — випадковий шум без структури. Якщо залишки мають патерн — модель недостатньо захопила сигнал.

Сезонність vs Цикл — часта плутанина

Підвищені продажі іграшок перед Новим роком — сезонність: відбувається щорічно у той самий час. Криза у будівництві 2008 року — цикл: тривалість і глибина були невідомі заздалегідь. Різниця критична для вибору методу: сезонність можна запрогнозувати через Holt-Winters, цикли — через SSA або ARIMA.

Адитивна декомпозиція часового ряду: оригінал = тренд + сезонність + залишки

Метрики точності прогнозу

Як порівняти, наскільки добре прогнозна модель? Існує кілька широко вживаних метрик — кожна має свої сильні сторони і сліпі плями. Важливо вибрати метрику, що відповідає бізнес-задачі.

MAE Проста (1/n) Σ |yᵢ − ŷᵢ| Легко інтерпретується (ті самі одиниці, що й дані) Однаково карає великі і малі помилки Коли всі помилки рівно «дорогі» для бізнесу
RMSE Популярна √(1/n Σ(yᵢ − ŷᵢ)²) Штрафує великі помилки сильніше — чутливий до викидів Важко інтерпретувати в %, залежить від масштабу Коли великі помилки особливо небажані
MAPE Зрозуміла (100/n) Σ |yᵢ − ŷᵢ| / yᵢ (%) Відсоток — легко пояснити бізнесу Проблема при yᵢ → 0; асиметрична Порівняння між товарами з різними масштабами продажів
sMAPE Симетрична 100 Σ |yᵢ−ŷᵢ| / (|yᵢ|+|ŷᵢ|) (%) Симетрична: однакова ціна за надмір і недостачу Складніша для пояснення; теж погана при нулях Коли важливо однаково карати надпрогноз і недопрогноз
MASE Нуль-стійка MAE / MAE_naive = (1/h Σ|eₜ|) / (1/(n−m) Σ|yₜ−yₜ₋ₘ|) Стійка до нулів; масштаб-інваріантна; порівнянна між рядами Менш інтуїтивна (не у %); потрібна навчальна вибірка для знаменника Переривчастий попит (zeros/sparse demand) — RetailPulse slow-movers
MASE — Mean Absolute Scaled Error (детально)
MASE = MAE_тест / MAE_наїв_train де MAE_наїв = (1/(n−m)) · Σₜ₌ₘ₊₁ⁿ |yₜ − yₜ₋ₘ| ← Seasonal Naive на train Інтерпретація: MASE < 1 → модель краща за Seasonal Naive MASE = 1 → модель на рівні Seasonal Naive MASE > 1 → модель гірша за тривіальний baseline Для несезонного ряду (m=1): знаменник = MAE Random Walk (Naive)
MASE — рекомендована метрика M-Competition (Makridakis et al.). Для RetailPulse зі slow-moving SKUs (багато нулів у продажах): MAPE/sMAPE дають безглузді значення (ділення на нуль або ≈200%), а MASE коректно рахується навіть при yₜ=0. Ціль: MASE < 0.8 (краще за Seasonal Naive на 20%+).

Базові моделі (baseline) — точка відліку

Будь-яка складна модель має перевершувати просту baseline. Перш ніж будувати ARIMA або SSA — порівняйте з базовими. Якщо SARIMA не краща за Seasonal Naive — щось пішло не так.

BaselineПрогноз ŷₜ₊ₕКоли підходитьКоли не підходить
Naive yₜ (останнє значення) Випадкове блукання, фінансові ціни Ряди з трендом або сезонністю
Seasonal Naive RetailPulse yₜ₊ₕ₋ₘ (той самий сезон минулого циклу) Сезонні ряди з плавним трендом Ряди зі змінюваною сезонністю або різкими трендами
Mean ȳ (середнє всього ряду) Стаціонарний ряд без пам'яті Майже будь-який реальний ряд
Drift yₜ + h · (yₜ − y₁)/(T−1) Рівномірний лінійний тренд без сезонності Сезонні або нелінійні тренди
Seasonal Naive як еталон для RetailPulse

Seasonal Naive каже: «продажі наступного тижня = продажі того ж тижня рік тому» (m=52 для тижневих даних). Це неймовірно проста модель, але вона автоматично враховує весь сезонний патерн. Для роздрібних продажів Seasonal Naive зазвичай дає MAPE 15–25%. Якщо ваша ARIMA дає 22% — вона не краща за «минулорічний календар». Мета RetailPulse: MAPE < 10% — удвічі краще за baseline.

Горизонт прогнозування та накопичення похибки

Точність прогнозу завжди падає зі збільшенням горизонту — це не дефект конкретної моделі, а фундаментальна властивість. Чим далі у майбутнє, тим більше накопичується невизначеність. Для RetailPulse горизонт 14 днів — прийнятний компроміс між точністю і корисністю для управління запасами.

Аналогія: метеорологія

Прогноз погоди на завтра — досить надійний. На 3 дні — ще корисний. На 10 днів — сильно наближений. На 30 днів — майже без смислу. Не тому що метеорологи погано рахують, а тому що система хаотична: дрібні похибки у початкових умовах швидко ростуть. З часовими рядами продажів — аналогічно.

Накопичення похибки прогнозу з горизонтом h
Random Walk ARIMA(0,1,0): Var[êₜ₊ₕ] = h · σ²ε → стандартна похибка ∝ √h AR(1) стаціонарний (|φ|<1): Var[êₜ₊ₕ] = σ²ε · (1 − φ²ʰ)/(1 − φ²) → насичується при h→∞ до Var[yₜ] = σ²/(1−φ²) MA(q): Var[êₜ₊ₕ] = σ²ε · (1 + Σⱼ₌₁^min(h-1,q) θⱼ²) → стає постійним при h > q (MA-пам'ять скінченна) ARIMA(p,d,q) загально: Var[êₜ₊ₕ] = σ²ε · Σⱼ₌₀^{h-1} ψⱼ² де ψⱼ — коефіцієнти MA(∞) представлення
Практичний висновок: для Random Walk і ARIMA з d≥1 невизначеність зростає необмежено — конус прогнозу розширюється з горизонтом. Для стаціонарного AR(p) невизначеність насичується на рівні Var[yₜ]. RetailPulse (14 днів = h=2 тижні): Var[êₜ₊₂] ≈ 2σ² для наївного прогнозу — звідси і зростання інтервалів прогнозування на графіках.
Q&A
Питання для самоперевірки
01Чим часовий ряд відрізняється від звичайної вибірки з точки зору статистики?
+
Класична статистика припускає, що спостереження незалежні (i.i.d. — independent and identically distributed). У часових рядах це припущення порушується: значення yₜ статистично залежить від yₜ₋₁, yₜ₋₂ тощо. Це означає, що стандартні тести (t-тест, F-тест) застосовувати некоректно без додаткових перевірок. Також порядок спостережень принциповий — зміна порядку знищує часовий сигнал.
02Коли використовувати адитивну, а коли мультиплікативну модель декомпозиції?
+
Адитивну — коли амплітуда сезонних коливань відносно стала, незалежно від рівня тренду. Наприклад, «продажі завжди зростають на 500 одиниць перед Новим роком». Мультиплікативну — коли амплітуда пропорційна рівню тренду: «продажі завжди зростають на 20% перед Новим роком» — тобто чим більший загальний обсяг, тим більший абсолютний сезонний стрибок. Перевірити: якщо на графіку залишків є зростаюча дисперсія — потрібна мультиплікативна модель.
03MAPE = 8% — це добре чи погано? Від чого залежить?
+
Це залежить від контексту. 8% MAPE для прогнозування обсягу виробництва нафти — чудово. 8% для прогнозування вузького SKU з нестабільним попитом (де «нуль» — нормальне значення) — може означати фактично безглуздий прогноз. Також MAPE асиметрична: похибка 200% (продали 1, прогнозували 3) і похибка 50% (продали 3, прогнозували 2) мають різне значення, хоча різниця в 2 одиниці. Оцінювати MAPE потрібно завжди у порівнянні з baseline (наприклад, наївний прогноз «як учора»).
04Як periodogram допомагає знайти сезонний лаг m перед побудовою SARIMA?
+
Periodogram — оцінка спектральної щільності S(ω): шукаємо частоту ω*, де S(ω) максимальна. Якщо ω* = 1/52 на тижневих даних — значить m = 52. Практично: у Python використовуємо scipy.signal.periodogram, шукаємо піки, конвертуємо частоту у кроки (m = 1/ω*). Для RetailPulse тижневий ряд дав піки на частотах 1/52 (річна сезонність) і 1/4 (місячна). Важливо: перед аналізом periodogram видаліть тренд першою різницею або STL-деТрендингом, бо потужний низькочастотний тренд «заглушує» сезонні піки. Після підтвердження піку через periodogram — використайте ACF/PACF на сезонних лагах (k = m, 2m, ...) для вибору P та Q.

Python m2_preprocessing.py
Пропуски (interpolate + сезонна медіана), IQR-відбір викидів, winsorizing, seasonal_decompose
import pandas as pd
import numpy as np
from statsmodels.tsa.seasonal import seasonal_decompose

series = weekly.copy()

# 1. Виявлення та заповнення пропусків
print(f"Пропуски: {series.isna().sum()} із {len(series)}")

# Лінійна інтерполяція (для коротких пропусків, 2-3 тижні)
series_filled = series.interpolate(method='time')

# Сезонна медіана (для тривалих пропусків)
for idx in series[series.isna()].index:
    w = idx.isocalendar().week
    vals = series[series.index.isocalendar().week == w].dropna()
    series_filled[idx] = vals.median() if len(vals) else series.median()

# 2. Виявлення викидів — IQR
Q1, Q3 = series.quantile(0.25), series.quantile(0.75)
IQR = Q3 - Q1
fence_lo, fence_hi = Q1 - 1.5 * IQR, Q3 + 1.5 * IQR
outliers = series[(series < fence_lo) | (series > fence_hi)]
print(f"Викиди (IQR): {len(outliers)}")

# Winsorizing — обрізання до меж
series_clean = series.clip(lower=fence_lo, upper=fence_hi)

# 3. Декомпозиція
result = seasonal_decompose(series_clean.dropna(),
                            model='additive', period=52)
total_var = result.observed.var()
for name, comp in [('Тренд', result.trend),
                   ('Сезонність', result.seasonal),
                   ('Залишки', result.resid)]:
    print(f"{name:12s}: {comp.var() / total_var:.1%}")
Python m2_metrics.py
MAE, RMSE, MAPE, sMAPE, MASE — функції + порівняльна таблиця Naive / Seasonal Naive / Mean
import numpy as np

def mae(y, p):   return np.mean(np.abs(y - p))
def rmse(y, p):  return np.sqrt(np.mean((y - p) ** 2))
def mape(y, p):  return np.mean(np.abs((y - p) / y)) * 100
def smape(y, p):
    return np.mean(2 * np.abs(y - p) / (np.abs(y) + np.abs(p))) * 100
def mase(y_te, p_te, y_tr):
    return mae(y_te, p_te) / np.mean(np.abs(np.diff(y_tr)))

# Train / test split (останні 13 тижнів — тест)
train = series_clean.iloc[:-13]
test  = series_clean.iloc[-13:]

# Baseline моделі
naive    = np.full(len(test), train.iloc[-1])
s_naive  = train.iloc[-52:].values[:len(test)]   # seasonal naive
mean_fc  = np.full(len(test), train.mean())       # mean forecast

print(f"{'Модель':16s} {'MAE':>8s} {'RMSE':>8s} {'MAPE%':>7s} {'MASE':>6s}")
print('-' * 52)
for name, pred in [('Naive', naive),
                   ('Seasonal Naive', s_naive),
                   ('Mean', mean_fc)]:
    y, p = test.values, pred
    print(f"{name:16s} {mae(y,p):8.1f} {rmse(y,p):8.1f} "
          f"{mape(y,p):7.1f} {mase(y,p,train.values):6.2f}")
